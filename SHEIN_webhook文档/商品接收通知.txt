商品接收通知
/product_document_receive_status_notice
自运营
全托管
SHEIN自营
半托管
POP
该消息用于通知 ERP SHEIN平台是否正常接收开发者的发品请求，消息接入方式可查看文档：webhook说明文档

消息属性
收起
数据类型	是否必须	描述
spu_name
string	否	
平台生成的SPU；对应发布接口的spu_name

version
string	否	
调用发品接口时返回的版本号

received_success
boolean	否	
SHEIN平台是否接收成功

document_details
object[]	是	
SHEIN平台接收的商品明细

document_sn
string	否	
商品审核的公文号

skc_name
string	否	
平台生成的SKC；对应发布接口的skc_name

sku_list
object[]	否	
SKU信息

seller_sku
string	否	
商家的SKU，对应发布接口的supplier_sku

sku_code
string	否	
平台生成的SKU；对应发布接口的sku_code

failed_reason
object[]	是	
平台接收失败原因

language
string	否	
语种

content
string	否	
失败的原因描述

FAQ
Q：Webhook消息发送失败了，平台会有重试机制吗？
2025-10-30
A：
如果平台将 Webhook 消息推送到开发者的回调地址后，开发者没有及时响应，会触发失败重推机制：非订单消息： 平台只会重试 1次。订单事件： 平台会重试 2次。重试时间： 第一次重试的时间大约为30分钟至1小时之间。如有进一步问题，请联系开放平台的工单支持进行咨询
