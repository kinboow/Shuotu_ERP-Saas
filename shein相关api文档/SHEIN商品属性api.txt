店铺查可选属性
POST
/open-api/goods/query-attribute-template
自运营
全托管
半托管
支持查询店铺可用属性。平台内不同店铺在各个类目下可用的属性不同，因此建议按店铺维度更新数据。同时因平台规则可能会变更，如增加销售属性，修改属性是否必填等，建议一周更新一次。

属性包括：商品属性、主销售属性（skc）、次销售属性（sku）、尺码表属性，属性规则复杂，可查看SHEIN商品属性介绍了解详细对接方式。

object[]	否

attribute_id
long	否	
属性id

attribute_name
string	否	
属性id的名称。常见的销售属性有颜色、尺码；常见的商品属性有产品特征、产品功能、是否需要电源等

attribute_is_show
int64	否	
属性是否会在消费者端显示；1-展示，2-不展示

attribute_type
int64	否	
属性类型。决定属性在商品发布接口中哪个字段内入参。

1-销售属性（在skc_list和sku_list入参）；2-尺寸属性（在size_attribute_list传参），3-成分属性（在product_attribute_list传参），4-普通属性（在product_attribute_list传参）

attribute_label
int64	否	
主销售属性（SKC属性标签）。只有主销售属性才可在skc_list中传参。

1-代表是主销售属性，主销售属性也可用于次销售属性; 0: 不是主销售属性。

attribute_mode
int64	否	
属性值的录入方式。

0 - 手工输入属性值，常见于尺寸属性，只可输入正整数，需在attribute_extra_value中入参；

1-下拉多选，即属性下可以有多个属性值，需在attribute_value_id中入参；

2-销售属性专用，下拉单选，需在attribute_value_id中入参；

3-下拉单选，需在attribute_value_id中入参

4-下拉多选+手工输入参数，需在attribute_value_id和attribute_extra_value中入参

attribute_input_num
integer	否	
属性值个数限制。

当attribute_mode=1时，最多可选的属性值个数，0代表无限制。

attribute_status
int64	否	
属性是否必填。 1-不可填写（属性可能已废弃）；2-选填；3-必填。必填属性如果不填，则会报错；

attribute_remark_list
integer[]	否	
属性标识。说明属性归属于什么场景。1:重要,2:合规,3:质量,4:关务

attribute_value_info_list
