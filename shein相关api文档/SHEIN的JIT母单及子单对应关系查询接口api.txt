JIT母单及子单对应关系查询接口
GET
/open-api/order/get-mothe-child-orders
全托管
SHEIN自营
POP
1.入参orderNos，selectJitMother必填，orderNos最多支持200个订单

2.info是个列表，列表实体必返回请求的订单号，motherOrChildOrders=空列表（不为null）的情况：

1）订单的供应商与请求的供应商不一致，

2）非JIt订单

3）查询的订单与selectJitMother不一致



实现逻辑：遍历请求订单编号，校验，如果订单是JIt母单 && 请求查询母单 则motherOrChildOrders为对应的子单信息

如果订单是JIt子单 && 请求查询非母单 且 则motherOrChildOrders为对应的母单信息

如果订单是JIt合单 && 请求查询非母单 且 则motherOrChildOrders为对应的母单信息



code=0返回成功，其他为异常，异常信息取msg

FAQ
Q：一次最多请求多少条数据？
2025-05-26
A：
采购单号支持批量查询，一次最多请求200采购单号。
Q：为何motherOrChildOrders数据为空？一般是什么原因？
2025-05-26
A：
日常的原因有：1）供应商ID出错，注意订单的供应商与请求的供应商不一致；2）非JIT订单；3）查询的订单号是母单，selectJitMother为子单；4）查询的订单号是子单，selectJitMother为母单；
Q：如何查询母单信息？如何查询子单信息？
2025-05-26
A：
a、查询母单下子单的基本信息：入参是母单订单号并且selectJitMother=1，则出参返回子单信息b、查询子单的基本信息：入参是子单或合单订单号并且selectJitMother=2，则出参返回母单信息
