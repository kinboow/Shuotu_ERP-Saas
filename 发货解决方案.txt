# 备货履约

## 一、适用角色
该解决方案适用于开发【代运营】或【SHEIN自营】应用模式的开发者，支持通过API/Webhook消息进行信息交互，完成备货履约业务。

## 二、业务介绍
业务介绍：备货单主要由SHEIN下单给到卖家，卖家备货给到SHEIN，SHEIN发货给到买家的业务模式

`SHEIN` → 下单备货 → `卖家` → 发货 → `SHEIN` → 发货 → `买家`

卖家侧流程：商品贴标签 → 打包 → 打物流面单 → 发货

## 三、概念&名词解释
1. 备货单：SHEIN平台给商家下的订单，有三种模式：
   - 模式1：急采：先按虚拟库存进行销售，有买家购买后再统一生成备货单的模式；
   - 模式2：直供：商家直接收到SHEIN通知备货的模式；
   - 模式3：JIT备货：SHEIN向商家下一批货但通过分批备货到货的模式；JIT单不需要履约，JIT备货需要履约时会生成备货模式的子单进行履约，该备货单的prepareTypeId=9；
2. 备货单发货：商家将备货单货品交付给到SHEIN使用的发货方式。发货有四种模式：
   - 模式1：通过SHEIN提供的合作服务商发货，其中服务商又分为以下两种：
     ① SHEIN合作的物流快速服务商
     ② SHEIN合作的区域车队服务商
   - 模式2：商家自行委托第三方服务商发货，其中服务商又分为以下两种：
     ① 商家自行委托物流服务商
     ② 商家自行委托的车队服务商
   - 模式3：自送/自提：商家自行送到平台仓或平台仓上门取件
   - 模式4：SHEIN上门查验：在商家的仓库分货给到SHEIN，SHEIN上门查验的方式
3. 打包方式：
   - 3.1 箱装发货：针对鞋子等易破碎、易挤压的商品，SHEIN会要求商家将商品打包后装入纸箱，此时适用于箱装发货，需要打印箱唛，箱唛发货一个采购单至少需要一个包裹
   - 3.2 包裹发货：针对服装等不易破坏的商品，此时适用于包裹发货，需要打印物流面单，商家一般为包裹发货，箱唛发货需要申请权限；
4. 打印：
   - 4.1 商品维度的信息打印：
     4.1.1 商品条码：送货到平台仓时，用以扫描入库的条码
     4.1.2 环保标签：部分国家地区要求商品必须要标注的环保信息
   - 4.2 发货单维度的信息打印：
     4.2.1 面单打印：物流运输相关的面单
5. 预估运费：使用SHEIN合作物流发货的场景下，查询商家所要承担的费用
   - 5.1 计算规则：同一个时效产品下，平台提供多个shein合作物流商家选择，当其中有给到推荐物流的情况下，平台以该推荐物流运费的一半为标准承担备货单运费，若无推荐则需商家自行承担全部运费；在部分场景平台会在承担的运费上再给到补贴；

## 四、备货单状态以及流转
代运营模式备货单状态流转图：
`开始` → 备货单下单给到商家 → 【已下单(2)】
- 已下单 → 加入到发货台（API无此操作） → 【发货中(3)】
- 发货中 → 成功创建发货单 → 【已送货(4)】
- 已送货 → SHEIN仓收到包裹 → 【已收货(5)】
- 已收货 → 商品质检不通过 → 【已退货(7)】 → 重新发货成功 → 【已下单(2)】
- 已收货 → 所有商品通过仓库质检 → 【已完成(8)】
- 已下单/已送货 → 取消发货 → 【已作废(10)】
- 已下单/已完成/已作废 → 自主作废成功/系统自动作废 → 【已作废(10)】

## 五、接口调用实践
1. 接口调用流程图：
- `开始` → 推送备货单信息 `/purchase_order_notice`（webhook获取） → 获取采购单信息 `/open-api/order/purchase-order-infos`（接口获取） → 打印商品条码 `/open-api/goods/print-barcode`（接口获取） → 是否需要打印环保标？
  - 是：获取全量耗材类型和耗材材质信息 `/open-api/goods-quality/environmental-label-rule/material-quality-tree-v2`（接口获取） → 打印合规标签 `/open-api/goods-compliance/label-print`（接口获取） → 发货基本信息查询接口 `/open-api/shipping/basic`（接口获取）
  - 否：直接进入发货基本信息查询接口
- 发货基本信息查询接口 → 选择商家发货地址 → 选择发货方式 → 是否使用SHEIN集成物流？
  - 是：物流产品查询 `/open-api/shipping/express-company-list-v2`（接口获取） → 预估运费查询 `/open-api/openapi-business-backend/purchase-estimated-fee`（接口获取） → 选择物流公司 → 查询收货仓信息 `/open-api/shipping/warehouse`（接口获取）
  - 否：直接进入查询收货仓信息
- 查询收货仓信息 → 创建发货单 `/open-api/shipping/orderToShipping`（接口获取） → 发货单维度打印 `/open-api/shipping/delivery-print-package`（接口获取） → 查询发货单列表 `/open-api/shipping/delivery`（接口获取） → 发货单变更通知 `/delivery_modify_notice`（webhook获取）

2. 实践描述：
   2.1 获取备货单信息：
   - 2.1.1 通过【采购单通知】Webhook通知获取最新的备货单；
   - 2.1.2 通过【获取采购单信息】接口获取备货单信息；
   2.2 打印商品条码：
   - 2.2.1 通过【商品打印条码】接口获取条码信息；
   2.3 打印环保标：
   - 2.3.1 通过【获取全量耗材标签规则】接口获取信息；
   - 2.3.2 通过【获取合规标签材质和耗材信息】接口获取信息；
   - 2.3.3 通过【批量打印环保标】接口获取信息；
   2.4 发货：
   - 2.4.1 通过【查询收货仓信息】接口获取收货仓信息；
   - 2.4.2 通过【发货基本信息查询接口】接口获取信息；
     2.4.2.1 送货方式：
     | 送货方式 | 是否支持的判断 | 获取具体的服务商信息 |
     | --- | --- | --- |
     | 上门送货 | deliveryTypeList.deliveryTypeValue=21 | / |
     | 自送/自提 | deliveryTypeList.deliveryTypeValue=41 | / |
     | SHEIN集成的物流快递服务商 | deliveryTypeList.deliveryTypeValue=1 | 通过【物流产品查询】接口获取信息 |
     | SHEIN合作的区域车队服务商 | deliveryTypeList.deliveryTypeValue=2 | expressCompanyList.isSupport=false的物流公司 |
     | 商家自行委托的物流服务商 | deliveryTypeList.isSupport=true且存在deliveryTypeList.deliveryTypeValue=1 | 【物流基本信息查询】接口获取信息 |
     | 商家自行委托的车队服务商 | deliveryTypeList.isSupport=true且存在deliveryTypeList.deliveryTypeValue=2 | 【物流基本信息查询】接口获取信息 |
   - 2.4.3 通过【查询收货仓信息】接口获取配送的仓库地址信息；
   - 2.4.4 通过【创建发货单】接口发货；
   2.5 打印发货单：
   - 2.5.1 通过【发货单维度打印】打印；
   2.6 获取发货单变更通知：
   - 2.6.1 通过【查询发货单列表】获取发货信息；
   - 2.6.2 通过【发货单变更通知】获取发货单报表；

## 六、常见问题
- Q：同一个商家能否同时存在多种备货单？
  A：同一时间存在多种备货时间
- Q：备货单会不会有多个SKC？
  A：备货单只会包含单个SKC
- Q：急采和备货单发货一样吗？
  A：相同模式的备货单一般情况下可以合并发货（除SHEIN预收货仓、安检标签值不一样等场景）；
  A：急采模式可以允许拆发货，但不可以少发（只发一部分，剩下的会被自动取消）；备货模式部分场景允许拆单发货，当部分发货后剩下不发的sku数量>20会生成一个新的备货单即拆单成功，当剩下的sku数量<=20会生成一个新的备货单即拆单失败；
- Q：备货单被退回如何处理？
  A：备货单被退回后，备货单状态会修改为已退货状态，商家可以针对备货单重新操作发货；已退货的备货单可重复的窗口日期，急采类型的退货时间+1天，备货类型的退货时间+14天；
- Q：备货单和发货单是1:1的关系吗？
  A：1个备货单对应1张发货单；备货发货的场景，备货单关联旧发货单和新发货单

## 七、接口以及webhook清单
| API名称 | 接口说明 |
| --- | --- |
| 订单-基本信息查询接口 | 获取采购单信息，包含SHEIN为您下的单独的采购单信息，包括采购订单和备货订单 |
| 发货单-获取采购信息 | 商家可以发货信息接口，获取商家发货信息 |
| 物流产品查询 | 获取SHEIN平台后可合作范围一致的物流方式（包含全国性物流和区域性物流） |
| 创建发货单 | 通过发货接口进行shein合作物流的托运，适用于全托管、shein自发货模式 |
| 预估运费 | 查询该笔订单的预估运费 |
| 打印商品条码 | 获取SHEIN对应的条码文件，适用于备货时需要贴SHEIN条码的商家 |
| 打印环保标签 | 获取SHEIN对应的标签文件 |
| 收货仓库查询 | 获取该笔发货单的目的地信息（平台收货仓信息） |
| 发货单维度打印 | 可以获取发货单面单，也可以按需选择是否打印包装箱的平台标签 |
| 发货单列表查询 | 可以通过发货单的时间和需要配送的仓库（单个/包裹）的平台相关要求 |
| 获取全量耗材打印信息（新） | 获取全量的耗材、材质和耗材类型（也包含外包装的种类和材质） |
| 合规标签打印 | 此接口支持打印合规相关的标签，包括：环保标、GPSR标签和环保标结合的标签 |

| Webhook清单 |
| --- |
| 采购单通知 `/purchase_order_notice` |
| 发货单变更通知 `/delivery_modify_notice` |
