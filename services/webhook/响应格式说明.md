# SHEIN Webhook响应格式说明

## 官方要求

根据SHEIN官方文档：
> **SHEIN会发送POST请求到您的回调地址，以验证您的回调地址可用，您必须返回code为2XX（200，201等）才会通过校验**
> **shein判断2xx返回码为成功。建议接到请求后异步处理，避免接口超时导致判断为失败**

## Java示例代码

```java
@PostMapping("/testOpenapiCallback")
public boolean testOpenapiCallback(HttpServletRequest request, @RequestParam("eventData") String eventData) {
    // 验签
    boolean verifySign = EventCallbackDemo.verifySignForHttp(request, SECRET_KEY);
    
    // 解密
    String decryptData = AESTools.decrypt(eventData, SECRET_KEY);
    
    // 异步处理
    executor.execute(() -> {
        handleRequest(decryptData);
    });
    
    // 返回验签结果
    return verifySign;
}
```

## 我们的实现

### HTTP响应

**POST /shein-full/callback**
- **状态码**: 200 (必须返回2xx)
- **响应体**: `'true'`
- **用途**: 
  1. SHEIN验证回调地址是否可用
  2. SHEIN推送事件通知

**注意**: SHEIN使用POST请求进行验证和推送，不是GET请求

### 处理流程

```
1. 收到SHEIN webhook请求
   ↓
2. 验证签名 (同步)
   ↓
3. 解密数据 (同步)
   ↓
4. 立即返回 200 + 'true'
   ↓
5. 异步处理业务逻辑
```

### 代码实现

```javascript
router.post('/callback', async (req, res) => {
  try {
    // 验签
    const isValid = verifySignForHttp(req.headers, APP_SECRET_KEY, requestPath);
    
    // 解密
    const decryptedData = aesDecrypt(eventData, APP_SECRET_KEY);
    
    // 异步处理
    setImmediate(() => {
      handleEvent(eventCode, data, openKeyId).catch(err => {
        console.error('[SHEIN全托管] 异步处理失败:', err);
      });
    });
    
    // 立即返回成功
    res.status(200).send('true');
    
  } catch (error) {
    // 即使出错也返回成功，避免平台重试
    res.status(200).send('true');
  }
});
```

## 响应规则

| 情况 | 状态码 | 响应体 | 说明 |
|------|--------|--------|------|
| 成功接收 | 200 | `'true'` | SHEIN认为推送成功 |
| 处理出错 | 500 | `'false'` | SHEIN会重试 |

**SHEIN重试机制：**
- 非订单消息：重试1次
- 订单事件：重试2次
- 重试时间：约30分钟至1小时
