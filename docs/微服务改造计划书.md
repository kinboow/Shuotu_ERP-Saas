# 跨境电商ERP系统 - 微服务改造计划书

## 一、项目概述

### 1.1 改造目标
- 将单体ERP系统改造为微服务架构
- 构建多平台统一同步引擎
- 实现服务独立部署、独立扩展

### 1.2 微服务划分

| 服务名称 | 端口 | 职责描述 |
|----------|------|----------|
| gateway | 5000 | API网关：统一入口、认证鉴权、路由转发、限流 |
| sync-engine | 5001 | 同步引擎：平台适配器、订单/商品/库存同步、Webhook接收 |
| oms | 5002 | 订单服务：订单CRUD、状态管理、发货单、订单统计 |
| wms | 5003 | 库存服务：多仓库存、库存同步、防超卖、库存预警 |
| pms | 5004 | 商品服务：SKU管理、平台商品映射、价格管理 |
| oss | 3001 | 文件存储服务：文件上传、签名URL、文件清理（已有） |
| misc | 5005 | 综合服务：用户管理、供应商、物流商、财务、标签打印、PDA |

## 四、数据库设计

### 4.1 数据库拆分策略

采用**逻辑隔离**方式，各服务使用独立Schema：

| Schema | 服务 | 主要表 |
|--------|------|--------|
| erp_oms | OMS | orders, order_items, shipments |
| erp_wms | WMS | inventory, warehouses, stock_logs |
| erp_pms | PMS | products, skus, sku_mappings |
| erp_sync | Sync | sync_tasks, sync_logs, platform_configs |
| erp_misc | Misc | users, suppliers, logistics_providers, finance_records |

### 4.2 新增表清单

| 表名 | 所属服务 | 说明 |
|------|----------|------|
| sync_tasks | sync-engine | 同步任务记录 |
| sync_logs | sync-engine | 同步详细日志 |
| platform_configs | sync-engine | 平台配置（密钥、同步参数） |
| sku_mappings | pms | SKU平台映射关系 |
| stock_logs | wms | 库存变动流水 |
| warehouses | wms | 仓库信息 |

## 五、服务间通信

### 5.1 通信方式

| 场景 | 方式 | 说明 |
|------|------|------|
| 同步调用 | HTTP REST | 需要立即返回结果的场景 |
| 异步事件 | Redis Pub/Sub | 事件通知，不需要返回结果 |
| 任务队列 | Redis List | 需要可靠投递的任务 |

### 5.2 事件定义

| 事件名 | 发布者 | 订阅者 | 说明 |
|--------|--------|--------|------|
| order.synced | sync-engine | oms | 新订单同步完成 |
| order.statusChanged | sync-engine | oms | 订单状态变更 |
| order.confirmed | oms | wms | 订单确认，需扣减库存 |
| order.cancelled | oms | wms | 订单取消，需释放库存 |
| inventory.changed | wms | sync-engine | 库存变动，需同步到平台 |
| product.synced | sync-engine | pms | 商品同步完成 |

## 七、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 数据迁移出错 | 高 | 分批迁移，保留回滚脚本 |
| 服务间调用延迟 | 中 | 合理使用缓存，异步处理 |
| 平台API变更 | 中 | 适配器隔离，快速响应 |
| 并发问题 | 高 | Redis分布式锁，乐观锁 |
