库存查询
POST
/open-api/stock/stock-query
自运营
全托管
SHEIN自营
半托管
POP
该API支持通过批量sku、skc、spu查询库存

请求参数（请求头）
收起
参数名称	数据类型	是否必须	描述
Content-Type
string	是	
x-lt-openKeyId
string	是	
访问用户隐私数据时的唯一权限标识 openKeyId
x-lt-timestamp
string	是	
请求时间戳，格式为时间转换为毫秒的值，也就是从1970年1月1日起至今的时间转换为毫秒，如：1583398764000
x-lt-signature
string	是	
接口签名
请求参数（请求体）
收起
参数名称	数据类型	是否必须	描述
skuCodeList
string[]	否	
SHEIN生成SKU 最多传100个，skuCodeList/skcNameList/spuNameList ，请确保三个参数有且只有1个不为空；

skcNameList
string[]	否	
SHEIN生成SKC，skuCodeList/skcNameList/spuNameList ，请确保三个参数有且只有1个不为空；
spuNameList
string[]	否	
SHEIN生成SPU，skuCodeList/skcNameList/spuNameList ，请确保三个参数有且只有1个不为空；

warehouseType
string	是	
仓库类型；1:查备货在SHEIN仓的库存/ 2:查半托管，自主运营模式的虚拟库存/ 3:查代运营（全托管）、SHEIN自营模式的虚拟库存

请求示例
复制
curl --location --request POST 'https://openapi.sheincorp.com/open-api/stock/stock-query' \
--header 'x-lt-signature: test' \
--header 'x-lt-openKeyId: test' \
--header 'Content-Type: application/json;charset=UTF-8' \
--header 'x-lt-timestamp: 1752570817402' \
--data-raw '{
    "skuCodeList": ["I1omh30jb5ld"],
    "warehouseType": 2
}'


错误码	错误信息	解决方案
00-E-MSC-0000	
服务器发生错误
系统异常，请重试
00-E-MSC-0002	
SKU、SKC、SPU这三者中只需要传一个值
该接口以下入参字段必传，且只能选择其中一个参数作为入参，否则无法查询；入参字段：skuCodeList、skcNameList、spuNameList
00-E-MSC-0004	
SKC下的SKU数量超过100，SPU下的SKU数量超过100，请按照SKU传值
以下入参字段的数量不能超过100，入参字段：skuCodeList、skcNameList、spuNameList
00-E-MSC-0006	
不允许该供应商操作
该接口仅限查询自主运营、半托管、全托管类型店铺的仓库
00-E-MSC-0009	
SKU数量超过100
skuCodeList中，sku数量不能超过100
00-E-MSC-0010	
SKC数量超过100
skcNameList中，skc数量不能超过100
00-E-MSC-0011	
SPU数量超过100
spuNameList中，spu数量不能超过100
00-E-MSC-0012	
仓库类型无效
warehouseType无效，请按照文档提供的枚举值进行传参
